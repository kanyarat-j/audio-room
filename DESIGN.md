# Design Reference: Thai Audio Installation Landing Page

## Project Overview

This project is a demo landing page for a Thailand-based company that sells, designs, and installs audio systems for organizations.

The website should explain the company's services clearly for non-technical customers such as business owners, purchasing teams, office managers, school administrators, hotel teams, restaurant owners, and IT staff.

This project is landing page only.

- No back-office
- No admin page
- No login
- No database
- No product CRUD
- No backend API
- No CMS

## Design Direction

The visual direction is inspired by `https://designsystemsrepo.ai/`.

The website should feel:

- Light-first
- Pastel
- Grid-based
- Playful-tech
- Modern B2B
- Friendly but still premium
- Clear and easy to scan

Avoid a dark corporate look. The page should not feel heavy, overly formal, or generic. It should feel like a modern audio solutions hub: approachable, organized, and trustworthy.

## Visual Tokens

### Colors

Use a light, clean palette.

- Background: soft white, pale blue, light lavender
- Primary text: dark charcoal
- Secondary text: cool gray
- CTA: clear blue
- Accents: purple, pink, mint, soft yellow

Suggested values:

- `#FFFFFF` for main surfaces
- `#F7F8FF` for soft page background
- `#EEF4FF` for pale blue sections
- `#F4EEFF` for light lavender accents
- `#111827` for primary text
- `#5B6475` for secondary text
- `#2563EB` for primary CTA
- `#A855F7` for purple accent
- `#EC4899` for pink accent
- `#2DD4BF` for mint accent
- `#FDE68A` for soft yellow highlight

### Components

Use UI elements that feel clean and friendly.

- Rounded cards
- Pill buttons
- Soft shadows
- Pastel outlines
- Dotted grid backgrounds
- Small status badges
- Friendly icon cards
- Floating visual panels

Cards should be easy to read and not overly decorative. Use visual hierarchy through spacing, type size, and color accents.

### Typography

Use clear, modern typography with strong Thai readability.

- Headlines should be large, confident, and simple
- Body text should be short and easy to understand
- Avoid overly technical wording
- Avoid decorative or overly playful fonts

## Page Sections

### 1. Header

Include:

- Company logo or placeholder company name
- Navigation links:
  - บริการ
  - กลุ่มลูกค้า
  - ขั้นตอนงาน
  - ผลงาน
  - ติดต่อ
- Primary CTA pill: `ขอใบเสนอราคา`

### 2. Hero

Goal: explain what the company does within a few seconds.

Suggested headline:

> ระบบเสียงองค์กร ที่ออกแบบให้ใช้งานง่ายตั้งแต่วันแรก

Suggested subheadline:

> บริการจำหน่าย ออกแบบ และติดตั้งระบบเสียงสำหรับห้องประชุม โรงแรม ร้านอาหาร โรงเรียน โรงงาน และสำนักงาน

Suggested CTA buttons:

- `ขอคำปรึกษาฟรี`
- `ดูบริการของเรา`

Visual direction:

- Light pastel 3D audio blueprint
- Floating speaker, microphone, and service cards
- Dotted grid background
- Soft blue/purple glow

### 3. Services

Show the main service areas as card-based content.

Recommended cards:

- ระบบเสียงห้องประชุม
- ระบบไมโครโฟนและประชุมออนไลน์
- ระบบเสียงประกาศ/เสียงตามสาย
- ระบบเสียงร้านอาหาร โรงแรม และโชว์รูม
- จำหน่ายอุปกรณ์เครื่องเสียง
- สำรวจ ออกแบบ ติดตั้ง และสอนใช้งาน

Each card should explain the business value, not only the equipment.

### 4. Solution Hub

This section should make the company feel organized and professional.

Recommended blocks:

- บริการติดตั้งครบวงจร
- ระบบเสียงสำหรับองค์กร
- อุปกรณ์ที่เหมาะกับงบประมาณและพื้นที่
- ดูแลหลังการขาย

Use large cards or grouped panels that feel like a curated service dashboard.

### 5. Industries

Show who the service is for.

Recommended industry cards:

- องค์กรและสำนักงาน
- โรงแรมและร้านอาหาร
- โรงเรียนและมหาวิทยาลัย
- โรงงานและคลังสินค้า
- ร้านค้าและโชว์รูม

### 6. Installation Journey

Show the workflow as a clear step-by-step timeline.

Recommended steps:

1. รับโจทย์
2. สำรวจหน้างาน
3. ออกแบบระบบ
4. เสนอราคา
5. ติดตั้งและทดสอบ
6. สอนใช้งานและดูแลต่อ

This section should reduce customer uncertainty by showing that the process is structured.

### 7. Portfolio Placeholder

Show 3-6 project cards.

Use placeholders until real project data is available.

Examples:

- `[ใส่รูปผลงานจริง] ระบบเสียงห้องประชุมองค์กร`
- `[ใส่รูปผลงานจริง] ระบบเสียงร้านอาหาร`
- `[ใส่รูปผลงานจริง] ระบบประกาศภายในโรงงาน`

Do not invent client names or project details.

### 8. Final CTA

Goal: encourage the visitor to contact the company.

Suggested headline:

> พร้อมวางระบบเสียงให้พื้นที่ของคุณหรือยัง?

Suggested subheadline:

> พูดคุยกับทีมงานเพื่อประเมินพื้นที่และเลือกโซลูชันที่เหมาะกับงบประมาณของคุณ

Suggested CTA buttons:

- `ติดต่อทีมงาน`
- `ขอใบเสนอราคา`

## 3D Background Concept

The 3D background should be an interactive audio blueprint.

It should help visitors understand that the company does not only sell speakers. The company designs a full audio system for a real space, from microphones and control equipment to speaker positions and sound coverage.

### Recommended 3D Elements

- Abstract speaker modules
- Microphone objects
- Amplifier or control unit block
- Room layout grid
- Sound wave paths
- Audio nodes placed around the space
- Connection lines between devices
- Floating service cards
- Small badges such as `เสียงชัด`, `ใช้งานง่าย`, `ติดตั้งครบวงจร`

### Motion Behavior

Use smooth, readable motion.

- Hero: camera lightly tilts over a pastel room-grid audio system
- Services: service cards float into position above the grid
- Installation Journey: signal lines connect from one audio node to the next
- Final CTA: all nodes connect into one complete audio system, then a soft pulse moves toward the CTA

### Non-technical Explanation

The 3D background is a moving visual scene behind the website content. It shows how an audio system connects across a real space, so customers can quickly understand that the company plans and installs the whole system, not just individual equipment.

### Implementation Guidance

If implemented later:

- Use Three.js or React Three Fiber
- Use simple low-poly or abstract objects
- Use pastel materials, soft shadows, and ambient lighting
- Keep particles minimal
- Keep text more important than the background
- Provide a lighter mobile version
- Provide a static fallback for reduced-motion or lower-performance devices

## Content Rules

- Thai language first
- Copy must be polished, natural, and easy to understand
- Write for non-technical customers
- Focus on outcomes, trust, and ease of use
- Avoid long technical specifications unless they are necessary
- Do not invent prices
- Do not invent dates
- Do not invent equipment brands
- Do not invent client names
- Do not invent contact details
- Use clear placeholders for unknown details, such as:
  - `[ใส่ชื่อบริษัท]`
  - `[ใส่เบอร์โทร]`
  - `[ใส่ LINE Official Account]`
  - `[ใส่รูปผลงานจริง]`
  - `[ใส่รายละเอียดผลงานจริง]`

## Out of Scope

The following are not part of this project unless explicitly requested later:

- Back-office
- Admin pages
- Product add/edit/delete
- Login or authentication
- User roles
- Database
- Backend API
- CMS
- Payment
- Booking system
- Inventory management

## Acceptance Criteria

The landing page should:

- Use Thai as the primary language
- Make the company's service clear within a few seconds
- Match the light pastel grid theme inspired by `https://designsystemsrepo.ai/`
- Feel modern, friendly, trustworthy, and B2B-ready
- Use card-based sections and clear CTA buttons
- Explain the installation workflow clearly
- Use the 3D background concept to support the audio-system story
- Remain readable and fast on mobile
- Avoid all back-office or admin-related features
